# Jobs & Workflow Allocation Specification

## 1. Overview
The **Jobs System** manages the allocation of tasks to simulated human employees (Citizens) and AI sub-agents. It tests the primary AI agent's organizational design, workflow optimization, and human resource management capabilities.

## 2. Core Capabilities

### 2.1 Role Definition
- Creates organizational structures (e.g., Engineering, Sales, Marketing, Legal).
- Defines the specific competencies and permissions required for each role.

### 2.2 Task Queuing & Execution
- Tasks generated by other systems (e.g., "Build Product X" from Crafting, or "Draft PR Statement" from Dialogue) are placed into departmental queues.
- The AI agent must optimize how these queues are processed, balancing load to prevent bottlenecks.

### 2.3 Hiring & Firing (HR Mechanics)
- **Recruitment**: Consumes time and capital to pull new Citizens from the global pool into the enterprise.
- **Performance Reviews**: The system tracks the output of each simulated employee. The AI agent must identify underperformers and decide whether to train (Education System) or terminate them.

## 3. Architecture

### 3.1 Work Capacity Model
Every simulated employee has a maximum bandwidth per tick. Assigning tasks beyond this bandwidth drastically increases the error rate and spikes burnout (Healthcare System).

### 3.2 Skill-Task Matching
Tasks have specific skill requirements. If an employee with Level 2 Python is assigned a Level 5 Python task, the completion time increases exponentially, and the failure probability approaches 100%.

### 3.3 Context Switching Penalty
Simulates the inefficiency of assigning too many disparate tasks to a single employee. Frequent context switching degrades overall throughput.

## 5. Integration Points
- **Citizen System**: Provides the underlying human entities that fill the roles.
- **Healthcare System**: Over-allocation leads to stress and burnout.
- **Education System**: Training programs consume employee bandwidth, temporarily reducing task output.
